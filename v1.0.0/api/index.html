Zettel.ZettelEntry — Type
ZettelEntry

A single bibliographic entry stored as a key, an entry type (e.g. "article"), and an ordered dictionary of BibTeX-compatible field names to their string values.

Fields

  • key::String: unique citation key (e.g. "Einstein1905")
  • entryType::String: BibTeX entry type in lower case (e.g. "article", "book")
  • fields::OrderedDict{String,String}: ordered mapping of field names to values
source
Zettel.ZettelLibrary — Type
ZettelLibrary

A collection of ZettelEntry objects, indexed by their citation keys.

Fields

  • entries::OrderedDict{String,ZettelEntry}: ordered mapping of citation keys to entries

Zettel.fetchFromCrossref — Method
fetchFromCrossref(doi; userAgent)

Fetch bibliographic metadata from the CrossRef REST API for the given doi and return a ZettelEntry. userAgent can be set to a custom string (recommended by CrossRef polite-pool guidelines).

Example

	entry = fetchFromCrossref("10.1002/andp.19053221004")

Zettel.writeBblFromAux — Method
writeBblFromAux(auxPath; libraryFiles = nothing, outputPath = nothing, style = "auto")

Generate a .bbl file from a LaTeX .aux file using a ZettelLibrary.

If libraryFiles is not provided, the function uses \bibdata{...} entries from the .aux file, resolving name.json (preferred) or name.bib relative to the .aux directory.

If style is "auto", the function uses \bibstyle{...} from the .aux when present, falling back to "plain" otherwise.

Returns a named tuple with outputPath, absent, and usedKeys.

Zettel.writeJsonLibrary — Method
writeJsonLibrary(lib, filename)

Serialise a ZettelLibrary to a JSON file at filename.

The file uses tab characters for indentation (one tab per nesting level). Each entry is stored as an object with "key", "type", and "fields" properties that mirror the corresponding BibTeX fields.
